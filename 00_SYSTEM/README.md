KENHU OS

Mission

Build a complete AI-powered affiliate business system that is organized, repeatable, and continuously improving.

The goal is to replace memory with documentation so every important discovery, workflow, prompt, and experiment is permanently recorded.

---

Core Principles

1. Document everything important.
2. Improve continuously.
3. Make decisions using real data.
4. Build reusable systems instead of repeating work.
5. Archive instead of deleting.

---

Folder Structure

00_SYSTEM

- System documentation

01_MASTER_STUDIOS

- Approved AI studios

02_MASTER_PROMPTS

- Approved prompts

03_WORKFLOWS

- Step-by-step operating procedures

04_PRODUCTS

- Product information and assets

05_DATA

- Performance tracking and experiments

06_TEMPLATES

- Reusable templates

07_ARCHIVE

- Old versions and completed work

---

KENHU Rules

- Every important discovery must be saved.
- Every approved prompt must have a permanent home.
- Every workflow must be documented.
- Every experiment should be recorded.
- Never rely on memory for important information.

---

Vision

Create a personal operating system that grows over time and becomes more valuable with every project, making future work faster, more consistent, and easier to improve.
